# Status and parity

AIKit is implemented by translating AI SDK semantics into Swift. The vendored AI SDK repository lives at `vendored/ai-sdk/` and is treated as the source of truth for expected behavior.

## What AIKit focuses on

- `generateText` and `streamText` behavior parity (multi-step tool loop, stop conditions, usage aggregation, errors)
- streaming event model that can represent tool input deltas and non-delta tool inputs
- type-safe structured outputs (`OutputSpec` + `ObjectSchema<T>`)

## Provider status

- `AIKitOpenRouter`: implemented (chat, completion, embeddings; streaming supported).
- `AIKitOpenAI`: public provider surface exists, but model implementations currently throw `AIKitError.notImplemented(...)`.

## Tests

AIKit uses a translation-first, test-driven workflow. Tests live under `Tests/` and mirror AI SDK scenario groupings where possible.

For `useChat` / `ChatStore` parity tracking, see `AIKIT_USECHAT_TRACKER.md` at the repo root.
