# Outputs and `OutputSpec`

AIKit models “what you want back” via `OutputSpec`:

- `Complete`: the final output type
- `Partial`: the partial output type (used by `partialOutputStream`)
- `responseFormat`: provider-facing hint (`ResponseFormat`)
- parsing hooks: `parseComplete` / `parsePartial`

## Built-in outputs

AIKitCore ships a small set of outputs under `Output`:

- `Output.text()` → `String`
- `Output.json()` → `JSONValue`
- `Output.object(_:schema:)` → decode JSON into your `Codable` type
- `Output.typedObject(_:)` → decode using `SchemaProviding` (macro-friendly)
- `Output.array(_:elementSchema:)` → `[Element]` with AI SDK-compatible wrapper shape
- `Output.choice(options:)` → `String` choice

## Typed objects with one schema representation

AIKit’s canonical schema container is `ObjectSchema<T>`, which wraps provider-facing `JSONSchema`.

The preferred authoring path is macros (`@AIModel`), which generate `SchemaProviding` and `ObjectSchema<T>` for a type. See [Schemas and macros](../02-foundations/04-schemas-and-macros.mdx).

Macro schema:

```swift
import AIKit
import AIKitMacro

@AIModel
struct Person: Codable, Sendable {
  @Field("Full name", minLength: 1, maxLength: 100)
  var name: String
}

let ai = AIClient(model: model)
let result = try await ai.generate(
  "Return a JSON object for a person.",
  output: Output.typedObject(Person.self)
)

let person = try result.output
```
