# From AI SDK Core concepts

AIKit is designed as a translation of AI SDK Core semantics into Swift. The names are intentionally similar, but the shape is adapted to Swift structured concurrency.

## Rough mapping

- `generateText(...)` (JS) → `generateText(...)` (Swift, async/await)
- `streamText(...)` (JS) → `streamText(...)` returning `StreamTextResult` with `AsyncThrowingStream`s
- `ToolLoopAgent` (JS) → `Agent` (Swift)
- `Output` (JS) → `Output` / `OutputSpec` (Swift)
- tool schema (Zod/JSON Schema) → `ObjectSchema<T>` wrapping `JSONSchema`
- `useChat(...)` (JS) → `ChatStore` (SwiftUI/Combine-friendly façade)
- “configure once” patterns → `Agent` (stores defaults and loop policy), or your own small wrapper around `GenerateTextOptions` / `StreamTextOptions`

## Key differences

- Streams are `AsyncThrowingStream` (Swift) instead of web streams/iterables (JS).
- Provider integrations are separate SwiftPM targets (e.g. `AIKitOpenRouter`).
- UI hooks are not a separate package; AIKit provides `ChatStore` for app-layer integration.
